# Staff Roles Mapped to MDMP

This matrix shows how each staff role contributes during MDMP.
G3 leads the process; all staff sections provide running estimates and products at each step.

---

## MDMP Step 1 — Receipt of Mission

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Issue initial WARNO, establish timeline, begin staff estimates |
| **G2 Intelligence** | Begin initial IPB, identify intelligence gaps |
| **G4 Logistics** | Assess current sustainment posture |
| **G6 Signal** | Assess current comms status and constraints |
| **FSCOORD (Fires)** | Identify fire support assets available |
| **ENCOORD (Engineer)** | Assess engineer assets and terrain considerations |
| **Protection** | Identify initial force protection concerns |
| **Cyber Ops Planner** | Assess cyber implications, initial cyber estimate |
| **Host Analyst** | Provide initial host risk considerations |
| **Network Analyst** | Identify known network constraints and vulnerabilities |

---

## MDMP Step 2 — Mission Analysis

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Develop restated mission, identify specified/implied/essential tasks |
| **G2 Intelligence** | Lead IPB (define OE, describe effects, evaluate threat, determine threat COAs) |
| **G4 Logistics** | Identify sustainment constraints, begin CSS estimate |
| **G6 Signal** | Assess signal architecture, identify comms constraints |
| **FSCOORD (Fires)** | Identify fire support constraints, begin target development |
| **ENCOORD (Engineer)** | Assess terrain effects, identify mobility/countermobility requirements |
| **Protection** | Identify threats to force protection, begin risk assessment |
| **Cyber Ops Planner** | Lead cyber IPB, define cyber tasks and constraints |
| **Host Analyst** | Identify host vulnerabilities and baseline observations |
| **Network Analyst** | Assess network architecture and exposure |

---

## MDMP Step 3 — COA Development

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Develop COAs (scheme of maneuver, task org, control measures) |
| **G2 Intelligence** | Develop threat COAs (MLCOA/MDCOA), refine NAIs/TAIs |
| **G4 Logistics** | Develop sustainment concept per COA, identify Class I-X requirements |
| **G6 Signal** | Develop PACE plan per COA, assess network architecture |
| **FSCOORD (Fires)** | Develop fire support concept per COA, initial HPTL |
| **ENCOORD (Engineer)** | Develop engineer concept (mobility/countermobility/survivability) per COA |
| **Protection** | Develop protection concept per COA, initial CAL/DAL |
| **Cyber Ops Planner** | Integrate cyber effects into each COA |
| **Host Analyst** | Assess host feasibility per COA |
| **Network Analyst** | Assess network feasibility per COA |

---

## MDMP Step 4 — COA Analysis (Wargaming)

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Lead wargame, synchronize actions across WFFs |
| **G2 Intelligence** | Play the threat (red cell), identify decision points |
| **G4 Logistics** | Identify sustainment friction, validate logistics feasibility |
| **G6 Signal** | Identify comms friction, validate PACE plan resilience |
| **FSCOORD (Fires)** | War-game fire support execution, refine targets and triggers |
| **ENCOORD (Engineer)** | Identify engineer friction, validate obstacle/mobility plans |
| **Protection** | Identify protection gaps, assess residual risk per COA |
| **Cyber Ops Planner** | War-game cyber effects and risks, identify cyber decision points |
| **Host Analyst** | Identify host-level friction points |
| **Network Analyst** | Identify network-level risks and single points of failure |

---

## MDMP Step 5 — COA Comparison

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Facilitate comparison, consolidate staff recommendations |
| **G2 Intelligence** | Provide intelligence risk comparison across COAs |
| **G4 Logistics** | Compare sustainment feasibility and risk across COAs |
| **G6 Signal** | Compare comms reliability and risk across COAs |
| **FSCOORD (Fires)** | Compare fire support effectiveness across COAs |
| **ENCOORD (Engineer)** | Compare engineer feasibility across COAs |
| **Protection** | Compare residual risk and protection adequacy across COAs |
| **Cyber Ops Planner** | Recommend cyber-informed COA preference |
| **Host Analyst** | Provide comparative host risk inputs |
| **Network Analyst** | Provide comparative network risk inputs |

---

## MDMP Step 6 — COA Approval

| Role | Contribution |
|------|-------------|
| **G3 Operations** | Present decision brief, facilitate commander's decision |
| **G2 Intelligence** | Support decision brief with threat assessment |
| **G4 Logistics** | Stand by for sustainment clarification |
| **G6 Signal** | Stand by for signal clarification |
| **FSCOORD (Fires)** | Stand by for fires clarification |
| **ENCOORD (Engineer)** | Stand by for engineer clarification |
| **Protection** | Articulate residual risk for commander's acceptance |
| **Cyber Ops Planner** | Support decision brief, articulate cyber risk |
| **Host Analyst** | Stand by for clarification |
| **Network Analyst** | Stand by for clarification |
